01

What is uv?

uv is an all-in-one Python package manager built in Rust by Astral (the team behind Ruff). It replaces pip, venv, pyenv, and pip-compile with a single, extremely fast binary that handles everything from Python installation to dependency locking.

Key insight: uv manages its own Python installs and virtualenvs automatically. You never need to activate a venv manually — just use uv run.
02

Installation (Windows)

The recommended way to install uv on Windows is via the official PowerShell installer. No admin rights required — it installs to your user profile.

Install via PowerShell

powershell
PS> powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
expected output
downloading uv 0.11.14 (x86_64-pc-windows-msvc) installing to C:\Users\oskars\.local\bin uv.exe uvx.exe uvw.exe everything's installed! To add C:\Users\oskars\.local\bin to your PATH, either restart your shell or run: set Path=C:\Users\oskars\.local\bin;%Path% (cmd) $env:Path = "C:\Users\oskars\.local\bin;$env:Path" (powershell)
PATH not updated yet? Either restart your PowerShell window, or run the $env:Path line shown above to apply it immediately in the current session.

Verify the install

powershell
PS> uv --version
expected output
uv 0.11.14 (3fdfdc7d4 2026-05-12 x86_64-pc-windows-msvc)
Self-updating: keep uv current by running uv self update at any time.
03

Project Initialization

Use uv init to scaffold a new project. uv creates a pyproject.toml, a README.md, and a pinned Python version. The --package flag controls which of two layouts you get.

Initialize an application (default)

Without --package, uv scaffolds a flat application: a runnable main.py at the root and a minimal pyproject.toml with no build system. Best for scripts and apps you run but don't distribute.

powershell
PS> uv init project_name --python 3.13 PS> cd project_name
expected output
Initialized project `project_name`

Initialize a package (--package)

With --package, uv scaffolds a src layout with an importable package, plus a [build-system] and a [project.scripts] entry point. This makes the project installable and distributable. Best for libraries or CLI tools you publish or install.

powershell
PS> uv init --package project_name --python 3.13 PS> cd project_name

Generated project structure

Application — flat layout with a runnable script:

project_name/ ├── pyproject.toml ← metadata (no build system) ├── .python-version ← pins Python version for the project ├── README.md ├── .gitignore └── main.py ← runnable entry script

Package (--package) — src layout, installable:

project_name/ ├── pyproject.toml ← metadata + [build-system] + [project.scripts] ├── .python-version ├── README.md ├── .gitignore └── src/ └── project_name/ └── __init__.py ← importable package code
.venv and uv.lock come later. Neither exists right after uv init — uv creates them on the first uv add, uv run, or uv sync. Commit uv.lock once it appears.
04

Adding Packages

uv add installs a package, adds it to pyproject.toml, and updates the lock file — all in one step.

Add packages

uv add takes one or more packages, space-separated.

powershell
PS> uv add opencv-python PS> uv add requests pandas numpy
expected output
Resolved 3 packages in 240ms Prepared 3 packages in 1.2s Installed 3 packages in 80ms + numpy==2.2.5 + opencv-python==4.11.0.86

Add a dev-only dependency

powershell
PS> uv add --dev pytest ruff

Dev dependencies go under [tool.uv.dev-dependencies] in pyproject.toml and are excluded from production installs.

Remove a package

powershell
PS> uv remove opencv-python
05

Python Version Management

uv manages Python installations itself — no pyenv or python.org installer needed. Versions are downloaded on demand and cached globally.

Install a Python version

powershell
PS> uv python install 3.13
expected output
Searching for Python versions matching: Python 3.13 Downloading cpython-3.13.3-windows-x86_64-none Installed Python 3.13.3 in 4.2s

Pin a project to a specific version

powershell — drop from 3.14 to 3.13
PS> uv python pin 3.13
expected output
Pinned `.python-version` to `3.13`

After pinning, re-run uv add <package> and uv will rebuild the environment against the new version.

List available Python versions

powershell
PS> uv python list
expected output
cpython-3.14.0a7-windows-x86_64-none <download available> cpython-3.13.3-windows-x86_64-none C:\Users\oskars\AppData\Roaming\uv\python\... cpython-3.12.9-windows-x86_64-none <download available>
06

Running Scripts

uv run executes a command inside the project's managed virtual environment — no activation step needed. It syncs the environment against the lock file before every run.

Run your main script

powershell
PS> uv run main.py
expected output
Using CPython 3.13.3 interpreter at: .venv\Scripts\python.exe Hello from my-project!

Run tools installed in the project

powershell — pytest, ruff, etc.
PS> uv run pytest PS> uv run ruff check .

Run a one-off tool without adding it to the project

powershell — uvx runs in a throwaway env
PS> uvx black .
No more activate/deactivate. uv run handles environment activation internally. You can still run .venv\Scripts\activate if you prefer an interactive session, but it is never required.
Quick Reference
uv init --python 3.13 Create a new project
uv add <pkg> Install & lock a package
uv remove <pkg> Uninstall a package
uv sync Sync env from lock file
uv lock --upgrade Upgrade locked versions
uv run main.py Run script in managed env
uv pip list List installed packages
uv python pin 3.13 Pin project Python version
uv python install 3.13 Download & install Python
uv add --dev pytest Add dev-only dependency
uv self update Update uv itself
uvx <tool> Run tool without installing
uv pip freeze Export installed versions
